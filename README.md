【目的】
忙しい朝の服選びを効率化し、前回特定の誰かに会った時と同じ服を着てしまう「服被り」を防ぐためのクローゼット管理システム。
【利用者の入出力】
入力： 洋服の画像（カメラ撮影またはファイル選択）、タグ情報（システム定義の選択式、またはユーザーが追加した自由入力のカテゴリ・色）、服被り防止のためのメモ（前回会った人の名前など）、プレビュー表示のための上下の服の選択。

出力： 登録された洋服の写真一覧、指定したタグによる絞り込み検索結果、色ごとの並び替え結果、選択したトップスとボトムスを並べたコーディネートプレビュー。

【制約】
プラットフォーム： Webブラウザのみ対応（スマホアプリ化は対象外）。

認証： ユーザー登録およびログイン機能は持たない。

データ保存： 登録された洋服データや画像は、利用端末のブラウザ内（ローカル領域）のみに保存される。ブラウザの容量制限を考慮し、画像登録時に自動でリサイズ（軽量化）を行う。

【受け入れ基準】
ログインなしで、ブラウザ上に洋服画像とタグ（カテゴリ・色・メモ）を登録・保存・編集・削除できること。

登録した洋服一覧を、指定したタグで絞り込み検索、および色ごとに並び替えができること。

一覧から「トップス」と「ボトムス」をそれぞれ選択し、2つの画像を画面上で並べて確認（プレビュー）できること。

ページをリロードしたり、ブラウザを一度閉じたりしても、登録したデータが保持されていること。
